SECTION 3 -- BUG ROOT CAUSE ANALYSIS
--------------------------------------------------------------------------------

  BUG 1 -- Overflow Flag (flag_ovf)
  -----------------------------------------------------------------------
  Location : out_reg always_ff block, line 245
  Buggy    : flag_ovf <= is_arith ? comb_carry : 1'b0;
  Fixed    : flag_ovf <= is_arith ? ovf_flag   : 1'b0;  // BUG FIX 1

  comb_carry  = add_out[32]           -- unsigned carry-out
  ovf_flag    = (~(A[31]^B_inv[31])) & (A[31]^add_out[31]) -- signed overflow

  These are orthogonal. Using carry as overflow gives:
    False positive: carry=1, no overflow (ADD -1+1, SUB 10-3, SUB 5-5)
    False negative: carry=0, overflow=1 (ADD 0x7FFFFFFF+1)
  Failures: FAIL 1, 2, 3, 4 | SVA: ast_ovf_equals_ovf_flag x4

  BUG 2 -- DIV/MOD Guard Signal (latent)
  -----------------------------------------------------------------------
  Location : result_mux always_comb, ALU_DIV and ALU_MOD case arms
  Buggy    : if (operand_b == 32'b0)
  Fixed    : if (b_gated   == 32'b0)  // BUG FIX 2

  b_gated is the power-gated operand actually fed to the divider.
  operand_b is the raw input port.
  For all 38 vectors needs_b=1 so they are identical -- bug is latent.
  Surfaces at gate level with active ICG or system-level power management.
  Failures: 0 visible | SVA: A9/A10 pass vacuously

  BUG 3 -- SRA Sign Extension (shift_fill)
  -----------------------------------------------------------------------
  Location : Unified barrel shifter, combinational assign, line 97
  Buggy    : assign shift_fill = 1'b0;
  Fixed    : assign shift_fill = (alu_op == ALU_SRA) ? operand_a[31] : 1'b0;  // BUG FIX 3

  shift_fill is replicated into vacated MSB positions of the 5-level
  right-shift tree. For SRA it must carry the sign bit; hardwiring to 0
  makes SRA identical to SRL for all negative inputs.
  Failures: FAIL 5, 6 | SVA: ast_sra_neg_signext x2
